## 1. システム概要

### 1.1 目的

訪問介護事業所における以下の業務をデジタル化・自動化する:

- ヘルパーのシフト管理
- 利用者（ケアクライアント）の情報管理
- 給与計算・明細書作成
- 法定書類の自動生成（AI活用）
- 実績データの取込・管理
- 書類スケジュール管理とコンプライアンス監視

### 1.2 対象制度

| 制度 | 説明 |
|------|------|
| 障害者総合支援 | 居宅介護、重度訪問介護、行動援護、同行援護 |
| 地域生活支援事業 | 地域の障害者向けサービス |
| 介護保険 | 高齢者向け介護サービス |
| 自費サービス | 制度外の有料サービス |

## 2. ユーザーと権限

### 2.1 認証方式

- Google OAuth 2.0（Supabase Auth経由）
- ホワイトリスト制: ヘルパーマスタに登録済みのメールアドレスのみログイン可
- 特別アカウント: `info@alhena.co.jp` は管理者固定

### 2.2 ロール

| ロール | 権限 |
|--------|------|
| admin | 全機能へのフルアクセス。ヘルパー管理、利用者管理、給与、書類、設定 |
| staff | シフト閲覧・編集、個人データの参照のみ |

### 2.3 個人シフトPWA

- ヘルパーごとにユニークトークンを発行
- `/personal/:token` でモバイル向け個人シフト画面を表示
- PWAとしてホーム画面にインストール可能
- オフライン閲覧対応

## 4. 機能要件

### 4.1 シフト管理

#### 4.1.1 シフト表表示
- 月単位のグリッド表示（ヘルパー × 日付）
- サービス種別ごとの色分け
- 月ナビゲーション（前月・次月・当月）
- リアルタイム同期（複数ユーザー同時編集対応）

#### 4.1.2 シフト入力
- インライン編集（サービス種別セレクタ付き）
- 開始時間・終了時間・利用者名・サービス種別の入力
- 複数スロット対応（/2, /3表示）

#### 4.1.3 一括操作
- 重複シフト自動除去
- 日付・月単位の一括削除
- 翌月反映（同一曜日パターンのコピー）
  - 事務・営業・会議・予定・休暇希望は反映対象外
- シフトのキャンセル・復元（5種のキャンセルステータス）

#### 4.1.4 シフト一括入力
- テキスト貼り付けからのパターン解析入力
- ヘルパー・月・サービス種別を選択後に一括作成

### 4.2 ヘルパー管理

#### 4.2.1 一覧・検索
- 名前検索、雇用形態バッジ表示
- 削除済みヘルパーアーカイブ

#### 4.2.2 ヘルパー詳細
- 基本情報（氏名、生年月日、連絡先）
- 雇用情報（雇用形態、入社日、部署）
- 報酬設定（時給制/月給制）
- 社会保険（健康・介護・厚生年金・雇用・労災）
- 税金設定（源泉徴収区分、扶養人数、住民税）
- 口座情報
- 資格一覧（取得日付き）
- 出勤テンプレート設定
- 月別手当（交通費、立替経費、手当、精算）

#### 4.2.3 個人シフトトークン
- PWA用ユニークトークン発行
- モバイルインストール対応
- ヘルパー個人が自分のシフトのみ閲覧

### 4.3 利用者管理

#### 4.3.1 一覧・検索
- 利用者ディレクトリ
- 名前検索・フィルタ

#### 4.3.2 利用者詳細（タブ構成）
- **基本タブ**: 連絡先、緊急連絡先、契約期間、障害支援区分
- **障害者総合支援タブ**: 受給者証、支援区分、支給量、責任者、計画相談、ケアプラン書類、同一建物減算、利用サービス
- **介護保険タブ**: 保険証項目（動的フィールド）
- **自費タブ**: 自費サービス追跡
- **書類タブ**: ケアプラン、手順書、アセスメント、モニタリング報告書

### 4.4 書類スケジュール管理

#### 4.4.1 ダッシュボード
- 利用者ごとの書類ステータス一覧（信号機カラー）
- 期限切れ・期限間近のアラート
- 一括生成・個別生成ボタン
- 生成進捗表示

#### 4.4.2 自動チェック
- 書類期限管理（計画書6ヶ月/モニタリング3ヶ月/手順書は随時）
- ステータス自動更新（active → due_soon → overdue）
- 契約日チェック（契約開始前の計画書作成促進）

#### 4.4.3 ケアパターン変更検知
- 前月 vs 当月の実績を比較
- 曜日×時間帯×サービスコードのパターン差分検出
- 変更時に手順書のステータスをdue_soonに更新

#### 4.4.4 自動再生成トリガー（3種）
1. **実績パターン変更**: CSV取込後にパターン差分検知 → 計画書自動再生成
2. **アセスメントアップロード**: 新規アセスメント登録後 → 計画書自動再生成
3. **目標期間満了**: 30日以内満了目標をAI判定 → change時は計画書自動再生成（24hスロットル）

#### 4.4.5 検証システム（8項目）
| チェック | 内容 | 重要度 |
|---------|------|--------|
| plan_before_contract | 計画作成日 < 契約開始日 | critical |
| helper_employment | ヘルパー入社日 ≤ サービス日 | critical |
| document_freshness | 3書類が生成済みかつ未失効 | critical |
| care_level_match | 障害支援区分が設定済み | warning |
| assessment_exists | アセスメント書類が登録済み | warning |
| plan_monitoring_period_match | モニタリング期限 ≤ 計画期間末 | warning |
| supply_amount_consistency | サービス種別が定義済み | warning |
| service_consistency | 実績のサービスコードが利用者設定と一致 | warning |

#### 4.4.6 計画変更要否チェック

**自動チェック（5種）:**

| チェック | 条件 |
|---------|------|
| 受給者証有効期限 | 残30日以下 |
| 障害支援区分変更 | 前回値と異なる |
| 支給量変更 | スナップショットが前回と異なる |
| モニタリング時期到来 | 期限日到来 |
| モニタリング変更要フラグ | 「あり」判定 |

**手動チェック（13項目）:**
- 生活変化: 卒業/入学、グループホーム移行、介護者変更、心身状態変化、入退院、その他
- サービス変化: 種類/量変更、利用者希望、その他

### 5.3 居宅介護計画書（care_plan）

#### 生成ルール
- アセスメント文書がある場合は内容に基づいて作成
- **用語の厳密一致**: アセスメントに記載された用語をそのまま使用（例: 「リハビリパンツ」を「おむつ」に変換しない）
- **ADL/IADLチェックリスト**: 移動、移乗、排泄、食事、入浴、更衣、整容、調理、洗濯、掃除、買い物、服薬管理

#### ブロック分割戦略（最重要）
- ブロック数 = 実績記録の一意の時間パターン数
- サービス種別ではなく時間パターンで分割
- 同一時間パターンの全サービスを1ブロックに統合

#### サービスコード判別
| コード先頭 | サービス種別 |
|-----------|-------------|
| 11xxxx | 身体介護 |
| 12xxxx | 家事援助 |
| 14xxxx | 重度訪問介護 |

#### 出力構造
```json
{
  "short_term_goal": "短期目標",
  "long_term_goal": "長期目標",
  "service1": {
    "service_type": "身体介護",
    "visit_label": "月〜金 午前の身体介護",
    "steps": [
      {
        "item": "援助項目名（15文字以内）",
        "content": "HH:MM 具体的手順（40-60文字）",
        "note": "個別の留意事項（40-60文字）"
      }
    ]
  },
  "service2": null,
  "service3": null,
  "service4": null,
  "short_term_goal_months": 3,
  "long_term_goal_months": 6,
  "period_reasoning": "期間設定根拠",
  "severity_level": "standard"
}
```

#### 目標期間のAI判定基準

| レベル | 障害支援区分 | 短期目標 | 長期目標 |
|--------|------------|---------|---------|
| standard | 区分3以下、状態安定 | 3ヶ月 | 6ヶ月 |
| moderate | 区分4、重度要素一部該当 | 2-3ヶ月 | 4-6ヶ月 |
| severe | 区分5-6、行動障害・医療的ケア | 1-2ヶ月 | 3-4ヶ月 |

#### GoalPeriod自動保存
- 計画書生成後、AI判定結果の `short_term_goal_months` / `long_term_goal_months` を使用
- 既存のアクティブGoalPeriodを `isActive: false` に更新
- 短期・長期の2レコードを `goal_periods` テーブルに自動作成

```json
          "issue": "課題・改善点（40-60文字）"
        }
      ]
    }
  ]
}
```

#### 計画変更連携
- `planRevisionNeeded === 'あり'` の場合、計画書スケジュールを overdue に更新
- 計画書再生成アクションがダッシュボードに表示される

## 6. 給与・経理システム

### 6.1 給与計算

#### 6.1.1 時給制

```
通常時間給 = 通常時間数 × (基本時給 + 処遇改善時給)
深夜時間給 = 深夜時間数 × (基本時給 + 処遇改善時給) × 1.25
同行給 = 同行時間数 × ¥1,200
事務給 = (事務時間数 + 営業時間数) × ¥1,200
総支給額 = 通常時間給 + 深夜時間給 + 同行給 + 事務給 + 各種手当
```

- 深夜時間帯: 22:00〜08:00（25%割増）
- 15分精度で通常/深夜を按分計算

#### 6.1.2 月給制

```
総支給額 = 基本給 + 処遇改善手当 + その他手当
```

- 出勤テンプレートに基づく勤怠自動生成
- 土日祝日・祝日自動除外（JP_HOLIDAYS 2025-2026データ）

#### 6.1.3 特別加算
- ヘルパー × 利用者の組み合わせごとに時給加算ルールを設定可能

### 6.2 社会保険計算

| 保険種別 | 料率 | 備考 |
|---------|------|------|
| 健康保険 | 5.12% | 標準報酬月額テーブル（¥58,000〜¥1,390,000、50等級） |
| 介護保険 | 0.80% | 40歳以上 or 明示的加入 |
| 厚生年金 | 9.15% | 標準報酬月額テーブル（¥88,000〜¥650,000、32等級） |
| 雇用保険 | 0.55% | 非課税通勤手当を含む |

- 端数処理: 50.5円以上は切り上げ、それ以下は切り捨て

### 6.3 源泉徴収税

#### 対応税年度
- 令和8年（2026年）: メイン
- 令和7年（2025年）: 後方互換

#### 税区分

| 区分 | 対象 |
|------|------|
| 甲欄 | 主たる勤務先（扶養控除等申告書提出者） |
| 乙欄 | 副業（申告書未提出者） |
| 丙欄 | 日雇い |

#### 2026年（令和8年）の変更点
- 基礎控除: ¥580,000/年（¥480,000から引上げ）
- 給与所得控除: 最低¥650,000/年
- 扶養人数（0〜7人以上）に応じた累進税率テーブル

#### 月次判定ロジック
- 1月支給: 翌年テーブル使用
- 12月支給（翌月実行）: 翌年テーブル使用
- その他: 当年テーブル使用

## 付録A: 書類生成フロー

```
1. データ収集
   ├─ 利用者情報（name, birthDate, careLevel, contractStart等）
   ├─ 実績記録（時間パターン抽出）
   ├─ 支給量（contract-authorized services）
   └─ アセスメント文書（あれば）

2. AI生成（Claude via Supabase Edge Function）
   ├─ テンプレート変数埋め込み
   ├─ システム指示（厳格ルール）
   ├─ アセスメントファイル添付（あれば）
   ├─ JSON形式で応答取得
   └─ 構造バリデーション・正規化

3. Excel出力（ExcelJS）
   ├─ ブロック別シート作成
   ├─ ヘッダー・フッター設定
   └─ 印刷設定（A4/横向き等）

4. 自動保存
   ├─ 利用者書類セクションに保存
   └─ スケジュール更新（次回期限計算）

5. 後処理
   ├─ GoalPeriod自動保存（計画書の場合）
   ├─ 検証実行（8項目）
   └─ 通知表示
```

## 付録B: 給与計算フロー

```
1. シフトデータ取得
   ├─ 対象月のシフト一覧
   ├─ キャンセル・削除済み除外
   └─ 時間0分のシフト除外

2. 時間集計
   ├─ サービス種別ごとに時間集計
   ├─ 通常/深夜の按分（22:00-08:00基準）
   └─ 15分精度の計算

3. 金額計算
   ├─ サービス別時給 × 時間
   ├─ 深夜割増（25%）
   ├─ 特別加算（ヘルパー×利用者ルール）
   └─ 手当加算（交通費、処遇改善等）

4. 控除計算
   ├─ 社会保険（標準報酬テーブル参照）
   ├─ 源泉徴収税（年度別テーブル参照）
   ├─ 住民税
   └─ その他控除

5. 差引支給額 = 総支給額 - 総控除額
```
